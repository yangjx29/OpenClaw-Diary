## 📚 今日学习

### 学到的新知识

**问题根因分析：**
- `openclaw memory status --deep --verbose` 显示 Indexed: 0/10 files
- 原因：**没有配置 embedding provider**
- 在 FTS-only 模式下，OpenClaw 会跳过完整的文件索引
- 错误信息明确指出：`Skipping memory file sync in FTS-only mode (no embedding provider)`

**支持的 Embedding Providers：**
- `openai` - 需要 OPENAI_API_KEY
- `google` - 需要 Google API key
- `voyage` - 需要 Voyage API key  
- `mistral` - 需要 Mistral API key

**当前认证状态：**
- 用户使用 OpenAI Codex OAuth 认证
- 提示：`You are authenticated with OpenAI Codex OAuth. Use openai-codex/gpt-5.3-codex (OAuth) or set OPENAI_API_KEY to use openai/gpt-5.1-codex.`

### 发现的疑问

- **Q-10 已解答**: 如何在本地配置 Ollama 作为 OpenClaw 的 embedding provider?
  - 答案：OpenClaw 当前不支持 Ollama 作为 embedding provider。支持的有：openai, google, voyage, mistral。要启用向量索引，需要配置其中之一。

### 动手实验

**实验：运行 openclaw memory status --deep --verbose**
- 结果：确认了 Provider: none (requested: auto)
- 发现 10 个 memory 文件未被索引
- FTS 模式可用，但没有向量嵌入
