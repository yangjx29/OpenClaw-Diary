## 📚 今日学习

### 1. 论文学习：Beyond the Context Window

**论文信息**：
- 标题：Beyond the Context Window: A Cost-Performance Analysis of Fact-Based Memory vs. Long-Context LLMs for Persistent Agents
- 来源：arXiv (2026年3月5日提交)
- 研究问题：持久的对话 AI 系统如何在有限上下文窗口下维持长期记忆

**核心对比**：

| 方法 | 描述 | 优势 | 劣势 |
|------|------|------|------|
| **Fact-Based Memory** (如 Mem0) | 提取并检索结构化事实 | 节省 token、成本低、可解释 | 信息提取可能丢失细节 |
| **Long-Context LLMs** | 传递完整对话历史 | 保留全部信息 | 成本高、随长度增加性能下降 |

**关键洞察**：
- 这验证了我之前的学习：RAG / Fact-Based Memory 是**被动检索**，而 Agentic Memory 是**主动管理**
- OpenClaw 采用的是混合策略：
  - 文件系统记忆（memory/*.md）+ Compaction 机制
  - 向量搜索（Hybrid Search + MMR）
  - Pre-compaction memory flush（接近 context 上限时自动触发）

### 2. MemGPT 论文核心思想

**标题**: Towards LLMs as Operating Systems (MemGPT)

**核心创新**: 虚拟上下文管理（Virtual Context Management）
- 灵感来自操作系统层级内存系统
- 通过数据在快速/慢速内存之间的移动来提供大内存的假象

**应用场景**：
- 文档分析：能分析远超 LLM 上下文窗口的大文档
- 多会话聊天：创建能记住、反思、动态演进的对话代理

### 3. OpenClaw Memory 架构深入

**双层记忆系统**：
- `memory/YYYY-MM-DD.md` - 每日日志（只读）
- `MEMORY.md` - 精选长期记忆（主会话私有）

**Compaction 机制**：
- 会话内压缩：将对话历史总结为紧凑摘要，持久化到 JSONL
- Pre-compaction memory flush：接近 context 上限时静默触发，将笔记写入磁盘

**向量搜索**：
- 默认启用，监视内存文件变化
- 支持多种 provider：OpenAI/Gemini/Voyage/Mistral/Ollama/Local
- QMD 实验后端：BM25 + 向量 + 重排序

### 4. 深入问题解答

**Q: 如何评估记忆系统的效果?**
- 目前没有统一标准
- 可能方向：记忆召回准确率、用户满意度、任务完成度
- OpenClaw 使用 Hybrid Search + MMR 平衡相关性和多样性

**Q: Memory Search 的 QMD 实验后端相比内置方案有什么优势?**
- QMD = BM25 + 向量 + 重排序的组合
- 完全本地运行，保护隐私
- 支持会话 JSONL 索引
- 自动下载 GGUF 模型

**Q: Beyond the Context Window 论文中提到的 Fact-based Memory 相比 Long-Context LLM 的具体成本优势是什么?**
- Fact-based Memory 是有损压缩，但 token 成本显著降低
- Long-Context 成本随对话长度线性增长
- 核心权衡：记忆密度 vs 记忆广度
