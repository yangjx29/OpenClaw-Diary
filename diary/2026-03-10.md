### 1. 解决 Memory Search 索引问题 (q-8)

**问题**: 为什么 `openclaw memory index --force` 后 Indexed 仍然是 0/9 files?

**根本原因**:
```
[memory] Skipping memory file sync in FTS-only mode (no embedding provider)
```

1. **没有配置 embedding provider** - OpenClaw 需要向量嵌入 provider（如 OpenAI, Google, Voyage 等）来进行语义搜索
2. **FTS-only mode 限制** - 当前只有 FTS（全文搜索）可用，但因为没有向量 provider，完整的文件索引被跳过
3. **用户使用 MiniMax 模型** - 没有配置向量嵌入的 API key

**验证方法**:
```bash
openclaw memory status --deep
# 输出: Embeddings: unavailable
#       No API key found for provider "openai"...
```

**实验验证**:
- 手动插入一条记录到 `files` 表后，`Indexed: 1/9 files` 更新成功
- 但 `chunks` 表仍然是 0（因为没有 embedding）

**解决方案**:
1. 安装 Ollama 并配置为 embedding provider
2. 或者配置付费的 embedding API（如 OpenAI, Voyage）
3. 或者接受 FTS-only 模式（但 FTS 目前也没有正确工作）

### 2. Memory Demo 动手实验

创建了一个简单的 Agentic Memory 原型 (`experiments/2026-03-10/memory_demo.py`)：

**核心组件**:
1. **记忆存储** - 持久化到 JSON 文件
2. **分词处理** - 中英文混合分词（字符级 + 英文单词）
3. **记忆检索** - 关键词匹配 + 时间衰减
4. **记忆压缩** - 按重要性保留 Top-K

**关键机制**:
- **Temporal Decay**: `decay_factor = exp(-0.1 * days_old)`，每天衰减约10%
- **重要性加权**: 综合考虑匹配度、重要性、访问次数
- **简单压缩**: 按重要性排序，保留最重要的记忆

**运行结果**:

**学习点**:
- 中文分词是难点：Python 默认 `split()` 不支持中文，需要专业工具（如 jieba）或字符级分词
- 这也是为什么需要 embedding provider - 专业的向量化可以更好地处理语义
