## 📚 今日学习

### 1. OpenClaw Memory 核心架构（深入理解）

**双层记忆系统：**
- `memory/YYYY-MM-DD.md` - 每日日志（追加模式），每天自动读取当天和昨天的内容
- `MEMORY.md` - 精选长期记忆，只在主会话加载（隐私保护）

**关键机制：**
- **Pre-compaction Memory Flush**: 当会话接近 context window 上限时，触发静默的记忆刷新轮次，将持久化笔记写入磁盘
- **自动触发条件**: `contextWindow - reserveTokensFloor - softThresholdTokens`
- 默认为静默模式（NO_REPLY），用户无感知

### 2. Compaction 机制详解

**Compaction vs Pruning:**
- **Compaction**: 总结并持久化到 JSONL（在磁盘上）
- **Pruning**: 仅修剪旧的工具结果（在内存中）

**配置参数：**
```json5
{
  compaction: {
    reserveTokensFloor: 20000,  // 保留 token 底限
    memoryFlush: {
      enabled: true,
      softThresholdTokens: 4000,
    }
  }
}
```

### 3. 向量搜索技术栈

**搜索模式：**
- Hybrid Search: BM25 + Vector 混合检索
- MMR (Maximal Marginal Relevance) 去重
- Temporal Decay 时间衰减（半衰期 30 天）

**Embedding 提供商：**
- OpenAI / Gemini / Voyage / Mistral
- 本地模式：node-llama-cpp + sqlite-vec

**QMD 实验后端：**
- BM25 + 向量 + 重排序
- 完全本地运行（Bun + node-llama-cpp）
