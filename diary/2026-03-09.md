### 1. OpenClaw 上下文压缩机制详解

今天深入阅读了 `docs/openclaw-context-compaction.md`，这是关于 OpenClaw 如何管理上下文窗口的详细文档。

**三道防线设计**：

| 防线 | 触发时机 | 操作位置 | 是否有损 |
|------|----------|----------|----------|
| 防线一：预截断 | 每次 LLM 调用（75% 阈值） | 内存 | 无损 |
| 防线二：Compaction | API overflow 错误 | 磁盘 | 有损 |
| 防线三：磁盘截断 | Compaction 失败 + 超大 toolResult | 磁盘 | 有损 |

**关键机制**：
- **预压缩记忆刷新**：接近 context 75% 时触发，将持久化笔记写入磁盘
- **Compaction**：将旧对话摘要化，持久化到 JSONL
- **摘要注入**：包含 Goal、Constraints & Preferences、Progress、Key Decisions、Next Steps、Critical Context

### 2. Memory Search 动手实验

今天尝试了 OpenClaw 的 memory search 命令：

```bash
openclaw memory status
openclaw memory index --force
openclaw memory search --query "memory"
```

**实验结果**：
- 当前 Provider: none (未配置 vector provider)
- FTS (Full-Text Search): ready
- Indexed: 0/8 files · 0 chunks（索引失败）

**分析**：
- 没有配置 embedding provider，所以向量搜索不可用
- FTS 虽然显示 ready，但索引没有真正工作
- 可能原因：memory 文件格式问题或权限问题

**待排查**：
